定义基础游戏对象管理

对于我们游戏中所有的物品（墙、蛇等），都需要我们在页面中完成动态的绘制。为了方便我们绘制这些物品，我们需要用到面向对象的思想，定义一个基类，然后通过继承基类来定义不同的对象。我们在基类中定义绘制的框架，再统一调用基类的绘制方法。

定义基类

这个基类是前端绘制的核心，我们只需要继承这个基类，然后定义一些绘制的函数即可

requestAnimationFrame

requestAnimationFrame 是一个 JavaScript 函数，用于请求浏览器在下一次重绘之前执行指定的动画函数。它通常用于创建流畅的动画效果，因为它会与浏览器的重绘周期同步，确保动画在每次屏幕刷新时更新。

由此，我们再绘制带动画效果的游戏物品时，只要在update方法中重写改变物品坐标颜色等即可

结语

  1. 为了整个游戏开发上的可扩展、可维护，创建了一个游戏的基类AcGameObject, 所有游戏对象都继承AcGameObject, 并且AcGameObject中包含了游戏对象的状态、渲染、更新、销毁等方法。

  2. 以requestAnimationFrame来渲染游戏对象，保证游戏的流畅性基本可以与用户浏览器帧率保持一致
