CodeSandBox之利用Docker执行动态代码

发表于2024-04-12|更新于2024-04-13|后端
|阅读量:

前言

CodeSandBox 是一个开源的，基于 Springboot 和 Docker 的在线代码沙箱。Github -> CodeSandBox

CodeSandBox 利用了 java-docker 的 api，将用户代码运行在 docker 容器中，与机器环境隔离，防止用户代码注入。

前期准备

在编写代码之前需要将 Docker 安装完成，同时制作一个轻量的可运行某个高级语言代码的镜像。

这里推荐使用 Alpine Linux, 这里只需要用到编译器以及一个 time 命令即可，time 命令用于统计程序执行时间和占用内存。

注意这里的 time 命令指的是 GUN Linux 的 time 命令，不是 Shell 内置的
其区别在于前者功能完善

后面将以 Java 作为样例进行描述，其他高级程序语言同理

要想远程创建Docker容器还需要对 Docker 进行一些配置

  1. 编辑Docker 配置文件 vim /lib/systemd/system/docker.service
  2. 在 ExecStart 后添加 -H tcp://0.0.0.0:2375

注意这是一种不安全的方式，因为这样方法不会对调用者进行校验

连接到DockerClient

编译并执行

在容器中完成编译与执行

  1. 将用户输入写入 Main.java 文件中 echo userinput > Main.java
  2. 编译 Main.java 文件 javac Main.java
  3. 执行 Main.java 文件 java Main
  4. 根据上一步，利用 time 统计程序执行时间和内存消耗并写入 consume.out 文件中 /usr/bin/time -f "%U:%K" -o consume.out java Main

注意，一定要给定容器内存限制和执行时间限制，防止恶意程序

获取输出

容器输出有两部分，一在控制台，可通过 docker logs <container_id> 查看；二在文件中，可通过 docker cp <container_id>:<file_path> <file_path> 拷贝到本地，再读取获取

总结

上述样例完成了基本的sandbox功能, 要想适应更多的代码可以将输入和输出都通过文件，而不是直接在 log 中读取，这样可以避免一些不必要的麻烦。

后续任务

  1. 加入任务池(队列)，对用户提交任务进行管理控制，防止大量提交导致Docker端压力过大。
  2. 适配更多语言，如Python、C/C++等
